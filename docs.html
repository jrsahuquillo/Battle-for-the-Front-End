HOW TO WRITE MY ROBOT?

Keep reading to know more about the javascript robotic API

Coding your robots is very easy and should be natural if you've ever had any Javascript experience.

This is the basic skeleton for your robot API

var Robot = function() {}; Robot.prototype.onIdle = function(robot) {}; Robot.prototype.onScannedRobot = function(robot) {}; Robot.prototype.onWallCollision = function(robot) {}; Robot.prototype.onHitByBullet = function(robot) {}; return Robot;

Robot events

.onIdle

The onIdle function is your robot game loop. This will be called every time your robot runs out of actions to do.

.onWallCollision

This method gets called whenever we hit a wall. The following variables are passed as argument up, down, left, right and one of them will be true depending on the direction in which you crashed.

.onScannedRobot

This event will be called whenever you see another robot. It's a very important event since it's usually when you want to fire.

.onHitByBullet

This event will be called whenever a bullet hits your robot.

Robot methods

The following methods are available inside your robot:

  • robot.ahead(units); Moves the given amount ahead.
  • robot.back(units); Moves the given amount backwards.
  • robot.stop(); Stops your tank for the given amount of time in units.
  • robot.turn(units); Rotates your robot the by the specified number of degrees.
  • robot.turnRight(units); Rotates your robot to the right.
  • robot.turnLeft(units); Rotates your robot to the left.
  • robot.rotateCannon(units); Rotates your cannon angle by the specified number of degrees.
  • robot.fire(); Fires your cannon. This function has a cooldown of 500ms before you can use it again.
  • robot.disappear(); Disappear your robot for two seconds. This method can only be used two times per fight.

The health of your robot can be accessed too:

  • robot.health Your tanks health

Example of usage:

Robot.prototype.onHitByBullet = function(robot) { if (robot.health < 4) { robot.disappear(); } };
